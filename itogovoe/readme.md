Проект реализован с использованием принципов объектно-ориентированного программирования и разделения ответственности.

Структура проекта:
```
project/
├── server.py        # HTTP-сервер и обработка запросов
├── task.py          # модель задачи (Task)
├── storage.py       # логика хранения и загрузки задач
├── validators.py   # валидация входных данных
├── tasks.txt        # файл хранения задач (создаётся автоматически)
└── README.md
```
## Task (task.py)

Класс Task описывает сущность задачи и содержит следующие поля:

- id — уникальный идентификатор задачи;
- title — название задачи;
- priority — приоритет (low, normal, high);
- isDone — статус выполнения задачи.

Также класс содержит методы:
- to_dict() — преобразование объекта в словарь для сериализации в JSON;
- from_dict() — восстановление объекта из словаря при загрузке из файла.

## TaskStorage (storage.py)

Класс TaskStorage отвечает за:
- хранение списка задач в памяти;
- генерацию уникальных идентификаторов задач;
- сохранение задач в файл tasks.txt;
- загрузку задач из файла при старте сервера.

Основные методы:
- add_task() — создание и сохранение новой задачи;
- get_all() — получение списка всех задач;
- complete_task() — отметка задачи как выполненной;
- save() — сохранение данных в файл;
- load() — загрузка данных из файла.
